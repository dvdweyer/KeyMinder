# Code of Conduct

## Our Standards

KeyMinder is a small open-source project. Participation is expected to be respectful and constructive. This means:

- Using welcoming and inclusive language
- Accepting constructive criticism gracefully
- Focusing on what is best for the project and its users

## Scope

This code of conduct applies in all project spaces: GitHub issues, pull requests, and any other forum where you represent the project.

## Reporting

If you experience behaviour that violates these standards, contact the maintainer at info@keyminder.app. All reports will be reviewed and addressed promptly and confidentially.
